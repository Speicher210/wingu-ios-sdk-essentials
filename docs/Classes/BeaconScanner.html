BeaconScanner

public final class BeaconScanner: NSObject

Layer of abstraction for CoreLocation ranging & monitoring iBeacons. This is build using Singleton principle. There will never be a need to use more than one instance of BeaconScanner to work with wingu beacons. By default regions to monitor and ranging are equal and default wingu regions apply if not specyfied otherwise.

  • rediscoverNotificationTime

    Time offset for notification. Whenever end-user rediscover the same beacon this will be check against last stored beacon. If passed more than time defined here it will send notification.
    By default this value is 1 hour (RediscoverNotificationTime.hourly). See RediscoverNotificationTime to see predefined values, but any value greater than 0 can be applied to this variable

    Declaration

    Swift

    public var rediscoverNotificationTime: RediscoverNotificationTime = .hourly
  • beaconLostOffset

    Property that prevent your beacons on the edge of discovery to jump back to lost state.
    Defined in seconds, for predefined values check BeaconLostTime.
    Value needs to be greater than BeaconLostTime.instant (2 seconds). In case of settings below this limit minimum limit will apply.

    Declaration

    Swift

    public var beaconLostOffset: BeaconLostTime = BeaconLostTime.instant
  • beaconFetchDataTime

    Variable is default content caching time.

    Note that setting this value low will cause extense battery and network data usage. When this value is set to 0 (.always) it will ask for content of a beacon every time whenever beacons is in range. That means every second when CLLocationManagerDelagate returns new beacons.

    It’s recommended to use it as high as possible depending on use-case.
    By default it’s normal (60 seconds).

    Declaration

    Swift

    public var beaconFetchDataTime: BeaconFetchDataTime = BeaconFetchDataTime.normal
  • shouldMonitorForBeaconsInBackground

    Property that indicates if you should monitor for beacons in background. Default is .sameAsRanging.
    This property will be overriden by direct call startMonitoringBeacons(regions: [CLBeaconRegion], locationManager: LocationManager?) and will be set to .custom whenever custom regions will be passed. It will be set to .none every time stop monitoring action will be performed.

    Declaration
