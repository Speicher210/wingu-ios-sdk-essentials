ContactComponent

public class ContactComponent : Component

It is a representation of info card in wingu portal. It may look like this:

It is possible to store all information in iOS default Contacts app, however there is need to add NSContactsUsageDescription in Info.plist file.

Note: iOS 11+ will raise any helpful flag and simply app will crash in case of missing key in Info.plist file mentioned above.

By default all links will be https ready from wingu portal if not specified otherwise.

  • extraInfo

    Additional information don’t feed in any other field. Since iOS nativly not support Google+ in Contact app, whenever there is Google+ filled and stored in Contacts app extra information in the end will be populated with Google+ link also.
