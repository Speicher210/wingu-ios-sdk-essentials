WinguLocations

public class WinguLocations : NSObject

Use WinguLocations to manage all triggers and locations inside wingu platform.
This class provide a way for managing zones and beacons in SDK.

Note: CLLocationManager shouldn’t be deallocated if notifications are required.

  • start()

    General starting method for ranging and monitoring beacons. It will start ranging beacons using default regions.

    Note: If there is a need to pass custom regions you should call startRangingBeacons(regions:locationManager:) instead

    It will also start monitor for geofences in foreground and in background.

    Start ranging beacon.

    Declaration

    public func startRangingBeacons(regions: [CLBeaconRegion], locationManager: LocationManager?)

    Parameters

    regions

    Collection of beacons regions, required. Pass empty array to scan for default regions.
    locationManager

    You can use your location manager, whenenever you pass nil CLLocationManager instance will be created for you.
  • stopRangingBeacons()

    Stop ranging beacons. This is async call for Apple Location Services, you may still receive some delegate call after calling stop ranging, but with response there will be always empty array of beacons. Note that this operation can take couple of seconds.
